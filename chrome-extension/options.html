AudioRouter Tabs

Every tab, its own speaker — standalone for Windows, macOS & Linux

What this extension does

Give each browser tab its own output device and its own volume (0–200%), simultaneously — a video call on your headphones while music plays to a speaker, or two YouTube tabs on two different outputs.

It's fully standalone: it works in any Chromium browser (Chrome, Edge, Brave, Arc) on Windows, macOS, and Linux — no companion app required. Windows users get what the built-in Volume Mixer can't do (per-tab, not just per-browser); Mac users can pair it with the AudioRouter app for macOS, which adds the system-wide per-app routing macOS itself lacks.

Device access

Chrome hides audio device names ("MacBook Pro Speakers", "AirPods", "Realtek Speakers"…) until an extension is granted media-device access, and only offers this through the microphone permission. AudioRouter never records or listens to anything — the permission is requested once, released immediately, and used purely to reveal the names of your output devices. Routing still works without it; you'd just see unnamed devices in the picker.

How it works

When you route a tab, the extension captures that tab's audio with Chrome's tabCapture API and replays it through a Web Audio pipeline — a gain node for volume, and AudioContext.setSinkId() to aim it at the device you chose. Capturing silences the tab's normal output, so audio plays exactly once, where you pointed it. Closing the tab or clicking ✕ instantly restores normal playback.

  • Routed tabs show Chrome's capture indicator — expected, that's the API.
  • To start routing a tab, be on it when you click the toolbar icon (a Chrome security rule). Already-routed tabs stay controllable from anywhere.
  • chrome:// pages and the Chrome Web Store can't be captured (Chrome policy).
  • Capture adds a few milliseconds of latency — imperceptible for music, fine for video.

Using the Mac app too?

If the AudioRouter macOS app has a whole-Chrome rule active, it captures all of Chrome's audio — including this extension's routed playback — and overrides per-tab choices. Remove Chrome's rule in the menu bar app while using per-tab routing.
